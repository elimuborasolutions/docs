Every user is assigned a role. Roles control which modules and actions are available. Roles are pre-configured — you assign them when creating or editing a user.

For a per-role guide, see the [Roles section](/roles/overview).

## How to assign a role
1. Open Users in the sidebar.
2. Select a user (or click New user).
3. Under Role, choose from the list.
4. Click Save.

## Role Summary
| Role | Access |
|---|---|
| Super Admin | Unrestricted access within your school |
| School Admin | School-wide operations and configuration |
| Principal | School-wide visibility, approvals, publishing |
| Deputy Principal | Same as Principal for daily operations |
| HOD | Manages a subject department |
| Class Teacher | Register and grading for assigned class |
| Teacher | Attendance and grades for assigned subjects |
| Bursar / Finance | Full Finance module access |
| Receptionist | Front desk records and basic enquiries |
| Librarian | Library inventory and book records |
| HR/Secretary | Staff records and school documentation |
| Driver, Chef, Nurse, Patron, Lab Tech | Scoped staff access |
| Guardian | Read-only view of own children |
| Student | Read-only view of own records |

<Info>
## Notes
- A user can hold one role at a time.
- Guardians and Students see only their own data.
- Only Super Admins can change role assignments.
</Info>

<Warning>
## Troubleshooting
- User can't see a module? Confirm role and that the module is active on your plan.
- Action blocked? Check whether the user owns the record (e.g. a teacher only marks their assigned classes).
</Warning>
